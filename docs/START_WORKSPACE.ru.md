# Автозапуск рабочей среды MedByClick

Скрипт `scripts/start-workspace.sh` поднимает всё, что нужно для начала
работы после перезагрузки Mac — без запуска команды из 10 AI-агентов.

## Что делает скрипт

1. Переходит в `~/medbyclick`.
2. Показывает `git status`.
3. Показывает `openclaw status` (если CLI `openclaw` установлен).
4. Открывает проект в VS Code.
5. Запускает `bun run dev` в отдельной вкладке Terminal.app.
6. Открывает `openclaw logs --follow` в ещё одной вкладке Terminal.app.
7. Готовит команду для запуска Claude Code (`cd ~/medbyclick && claude`) и
   копирует её в буфер обмена — но **не запускает** её сама.

Бизнес-логику проекта скрипт не трогает — только открывает инструменты.

## Как запустить

```bash
cd ~/medbyclick
./scripts/start-workspace.sh
```

При первом запуске macOS может попросить разрешить Terminal.app
управлять другими окнами (System Events / Automation) — это нужно для
открытия вкладок с dev-сервером и логами. Разрешите один раз.

## Важно: AI-команда из 10 агентов не запускается автоматически

Скрипт специально **не вызывает** `scripts/start-ai-team.sh`. Когда
будете готовы запустить всех 10 AI-агентов, сделайте это вручную:

```bash
./scripts/start-ai-team.sh
```

## После запуска скрипта

- Вставьте скопированную команду (`cd ~/medbyclick && claude`) в любой
  терминал, чтобы открыть Claude Code.
- Вкладка с `bun run dev` — это dev-сервер сайта.
- Вкладка с `openclaw logs --follow` — живые логи OpenClaw Gateway.
