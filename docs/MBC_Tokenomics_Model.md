## 3. Burn Mechanics

### Transaction Burn
- **2% of every MBC payment is burned**
- Permanently reduces total supply
- Deflationary pressure increases with platform usage

## 8. Anti-Whale Protection Parameters

### Current Settings (adjustable by owner)

| Parameter | Value | Purpose |
|-----------|-------|---------|
| Max Transaction | 1,000,000 MBC (1%) | Prevents large single dumps |
| Max Wallet | 3,000,000 MBC (3%) | Prevents whale accumulation |
| Cooldown | 30 seconds | Prevents bot manipulation |
| Owner Exempt | Yes | Owner can distribute freely |
| Adjustable | Yes | Owner can change via setLimits() |
| Removable | Yes | Owner can disable via removeLimits() |

### When to Adjust Limits

- **Increase maxTx** when daily volume exceeds 10x initial, to allow larger legitimate trades
- **Increase maxWallet** after CEX listings (CEX hot wallets need higher limits)
- **Remove limits** only after token is mature and widely distributed (Year 3+)
- **Never remove** if top 10 wallets hold >50% of circulating supply

## 9. Dual-Chain Economics

### Why Two Chains

| | Ethereum | BNB Chain |
|--|---------|-----------|
| **Use case** | Large payments, DeFi, staking | Daily payments, micro-tx |
| **Avg. tx size** | $100-5,000 | $5-100 |
| **Gas cost** | $5-50 | $0.05-0.20 |
| **Gas % of tx** | 1-5% | 0.05-1% |
| **DEX** | Uniswap | PancakeSwap |
| **Target user** | Investors, stakers | Patients, daily users |

### Supply Split Across Chains

| Chain | Allocated Supply | Purpose |
|-------|-----------------|---------|
| Ethereum | 60,000,000 MBC (60%) | Primary chain, staking, governance |
| BNB Chain | 40,000,000 MBC (40%) | Payments, rewards, daily operations |

Bridge allows rebalancing based on demand.
