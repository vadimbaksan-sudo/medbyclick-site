# LEGAL_BRIEF.ru.md — краткое резюме `docs/LEGAL_BRIEF.md` на русском

**Это не отдельный документ, а русскоязычная навигация по `docs/LEGAL_BRIEF.md`.**
Сам бриф (v1.1) остаётся на английском — он адресован юрфирмам-кандидатам
(Fieldfisher, DLx Law, MME Legal, Hogan Lovells, Sygna Partners) и должен уйти
им как есть. Здесь — только суть, чтобы не читать 375 строк по-английски.
Актуальность поддерживается вручную: при следующем существенном изменении
брифа этот файл нужно обновить отдельно.

## Что это за документ

`docs/LEGAL_BRIEF.md` — бриф для внешних юристов по крипто-праву. Описывает
токен MBC и просит: (1) заключение по классификации токена (MiCA, EU
Prospectus Regulation), (2) рекомендацию по юрисдикции для юрлица, (3) шаблон
SAFT/Token Warrant, (4) ревью MiCA White Paper перед подачей.

## Part 0 — Готовность к найму (самое важное, новое в v1.1)

Юридическое содержание брифа готово к отправке уже давно. Не хватало не
анализа, а организационной части. Блокеров пять, и три из них — **совместные
решения Марины и Вадима**:

1. **Выбор фирмы** из списка кандидатов — совместное решение (retainer =
   финансовое обязательство компании).
2. **Кто подписывает** engagement letter — юрлица ещё нет, подписывают
   фаундеры лично — совместное решение.
3. **Из каких средств оплачивается retainer** — treasury и выручки от продажи
   токена ещё нет, речь о личных деньгах фаундеров — совместное решение.
4. Шаблона NDA пока нет (нужен до передачи whitepaper/tokenomics/audit).
5. Поля с личными данными фаундеров в брифе — ещё плейсхолдеры; их фаундеры
   должны предоставить юристам сами после NDA, это не решение, а действие.

**Что НЕ является блокером** (то есть не нужно решать до начала контактов):
нерешённая юрисдикция, нерешённая классификация токена, неизвестный бюджет на
юруслуги — это как раз то, за ответом на что бриф и отправляется юристам.

Трекер контактов с фирмами — Part 0.3 английского файла, пока пуст (ни с кем
не связывались).

## Открытые юридические вопросы по разделам (кратко)

- **Part 2 — модель токена:** механика сжигания (burn) 40/40/20 и
  распределение 20% докторам — главный риск-флаг: может ли это трактоваться
  как profit-sharing по MiCA / Howey Test.
- **Part 3 — токеномика:** вопрос к юристам, не создаёт ли зарплата
  фаундеров из treasury (funded от продажи токена) риск характеристики
  продажи как investment contract.
- **Part 4 — продажа токена:** какой инструмент (SAFT / Token Warrant /
  Simple Token Purchase Agreement), нужна ли регистрация в FCA/AMF/BaFin,
  можно ли принимать инвестиции до регистрации юрлица через escrow.
- **Part 5 — классификация по MiCA:** наша самооценка — utility token;
  риск — вторичный рынок на PancakeSwap сразу после TGE может эту
  квалификацию подорвать. Howey Test для США: вероятно не проходит (что
  означает потенциальный статус security в США) — отсюда исключение US-лиц.
- **Part 6 — юрлицо:** кандидаты — BVI, Cayman, ADGM/DIFC (Дубай/Абу-Даби),
  Сингапур. Итоговое решение — совместное (Joint), сам бриф просит у юристов
  только рекомендацию.
- **Part 7 — treasury/мультисиг:** Gnosis Safe 3-из-5; вопрос к юристам — не
  создаёт ли мультисиг-казна признаки "collective investment scheme".
- **Part 8 — смарт-контракт:** до деплоя нужно юрлицо, протестированный
  Gnosis Safe, заключение по классификации, ревью vesting-контрактов юристами;
  есть технические находки (pause-функция на EOA, отсутствие событий у
  setExcluded(), setLimits() без таймлока), которые нужно устранить до
  деплоя.
